上新了！我的饭
北京交通大学食堂信息共享平台
时间 2025 — 进行中
角色 独立开发
技术栈 Python Flask · SQLite · Bootstrap 5
部署 PythonAnywhere

项目背景

北京交通大学食堂菜品丰富，但同学们选菜常靠运气、踩雷时有发生。「上新了！我的饭」希望为校园师生提供一个菜品浏览、评分评论、社区交流和三餐打卡的一站式平台，让每一次吃饭都有据可依。

目前已收录 8 个食堂、63 个档口、1,400 道菜品，已上线运行。

技术架构

整体采用服务端渲染（SSR）架构——Python Flask 通过 Jinja2 模板引擎直接渲染页面，前端仅处理少量交互逻辑（搜索联想、图片预览等），不引入前端框架，保持轻量和低依赖。

层级 技术选型
前端 HTML + CSS + JavaScript + Bootstrap 5
后端 Python Flask（模块化蓝图路由） + Jinja2
数据库 SQLite（10 张表）
认证 Flask-Login + bcrypt 密码加密
安全 CSRF Token 防护、session 管理
部署 PythonAnywhere（WSGI）

数据规模

食堂 楼层 菜品数 代表档口
东区食堂 一层 292 交大小笼包、重庆小面、炸鸡汉堡
学活食堂 三层 558 各类风味档口
学生四食堂 一层 90 减脂餐窗口
明湖食堂 两层 239 基本伙、面食、风味档口
西区食堂 一层 75 基本伙窗口

核心功能

食堂 → 档口 → 菜品三级浏览：食堂详情展示营业时间与实时人流量，菜品支持五星评分（一人一票可修改）与 500 字文字评论。16 个品类配有 AI 生成的手绘水彩风格图片。

智能首页：基于「评分×评论数」的推荐算法展示今日推荐，支持特价高亮（原价划线对比）、本周上新轮播，以及根据北京时间自动判断早/午/晚餐时段的人流量提示。

社区：用户可发帖（文字 + 最多 9 张图片 + 关联菜品），支持点赞、评论，发帖时菜名模糊匹配数据库。热门帖子 TOP 10 按点赞数排序。

三餐打卡：早餐 +3 分 / 午晚餐 +2 分的积分体系，打卡日历展示当日各餐状态，积分排行榜 TOP 10，随机打卡语录 10 条。

管理后台：数据统计面板、用户管理、违规内容删除、特价菜品设置。

设计亮点

时区处理：自定义 beijing_now() 函数，不依赖服务器本地时区，确保餐次提示、打卡日期、人流量判断在全球任意服务器上都能正确显示北京时间。

图片回退机制：get_dish_image() 三级回退——菜品自有图片 → 品类代表图 → 占位图标，保证任何菜品都有视觉展示。

模块化路由：认证、主路由、社区、打卡、管理后台各自独立 Blueprint 模块，app.py 作为应用工厂负责组装，结构清晰。

响应式布局：PC 端顶部导航 + 移动端底部 TabBar，适配校园实际使用场景（手机为主）。
