Memolog
把真实经历沉淀为可检索、可复用的个人记忆库
时间 2026.06
角色 独立全栈开发
前端 Next.js 16 + React 19 + TypeScript
样式 Tailwind CSS 4 + shadcn/ui
后端 Supabase（PostgreSQL + RLS）
AI DeepSeek API

项目背景

学生和初入职场的人常常面临一个问题：做过的事情散落在聊天记录、备忘录和记忆里，真正需要写简历、准备自我介绍时却想不起细节。Memolog 不是通用聊天工具，而是一个面向个人用户的"第二大脑"——随手记录经历，AI 自动整理为结构化记忆，并在需要时从真实记忆中生成内容。

技术架构

前后端分离的现代化全栈架构。前端基于 Next.js 16 的 App Router 构建，后端依托 Supabase 的 PostgreSQL 数据库与 Row Level Security（RLS）策略处理用户认证与数据隔离，AI 能力通过服务端 API Route 集成 DeepSeek，密钥不暴露到浏览器端。

层级 技术选型
框架 Next.js 16 + React 19 + TypeScript
UI Tailwind CSS 4 + shadcn/ui / Base UI
数据库 Supabase PostgreSQL（3 张表 + RLS 行级安全）
认证 Supabase Auth + 邀请码
AI DeepSeek API（服务端 Route，密钥不外泄）
部署 Vercel

核心功能

模块 说明
AI 整理记忆 输入一段自然语言经历，AI 提取日期、分类、标题、成果、内容、情绪与标签；保存前可逐项修改
日期与阶段 支持单日和阶段性经历；自定义日历可按月或按年切换
记忆库 按关键词、分类、时间筛选；支持查看、编辑和删除单条记忆
个人分类 在"档案"中自由添加或删除分类，记录、筛选与图谱会同步使用
记忆图谱 按分类、经历和标签展示关联，帮助发现经历之间的联系
AI 生成 基于个人档案和真实记忆生成简历、自我介绍或自定义内容
生成历史 自动保存生成内容，AI 为新记录提炼短标题作为索引；正文支持 Markdown 渲染，原始需求可折叠查看

数据模型

Supabase 初始化脚本创建 3 张表，每张表均启用 RLS 行级安全策略，用户只能访问自己的数据：

表名 用途
memories 用户记忆（日期、分类、标题、成果、内容、情绪、标签、阶段结束日期）
profiles 个人档案和自定义分类数组
generated_histories AI 生成历史（短标题索引 + Markdown 正文 + 原始需求）

项目结构

路径 职责
app/api/claude/ AI 提炼与生成接口（服务端 Route）
app/dashboard/ 记忆库与图谱
app/generate/ 简历、自我介绍与自定义生成
app/memory/[id]/ 单条记忆详情与编辑
app/profile/ 个人档案与分类管理
components/ 登录态外壳 / 记忆表单卡片图谱 / 复用 UI
lib/ Supabase、AI、日期和数据访问封装

安全与隐私

RLS 行级安全：数据按 Supabase 用户 ID 隔离，数据库层面限制每条记录只有创建者可读写。

密钥隔离：DeepSeek API Key 仅在服务端 Route 中使用，使用 DEEPSEEK_API_KEY 环境变量（不带 NEXT_PUBLIC_ 前缀），确保不暴露到浏览器端。

邀请码注册：可选配置 INVITE_CODE 环境变量，限制注册范围。
